## Завдання 
### Тематика: Книгарня

Спроектуйте базу даних для книгарні, для управління книгами, клієнтами та замовленнями.

### Підготовка схеми бази даних

Необхідні сутності
- Клієнти (ім'я, телефон, email)
- Книги (назва, рік видання, ціна, основний жанр)
- Автори (імʼя, національність, рік народження)
- Замовлення (час замовлення, статус, клієнт)
- Позиції замовлення (позиції в кожному замовленні з кількістю)

### OLTP запити

2 INSERT запити до будь-яких таблиць

2 UPDATE запити до будь-яких таблиць

2 DELETE запити з будь-яких таблиць

2 прості SELECT:
- Знайти всі книги конкретного автора
- Отримати деталі замовлення N (позиції, кількість, ціна, сума по кожній позиції)

### OLAP запити

- Топ-10 книг за загальною виручкою за останні 12 місяців
- Для кожного клієнта знайти його замовлення з номером по порядку (1, 2, 3, т.д.)
- Знайти авторів, чия середня виручка на книгу вища за середню по всіх авторах
- Знайти авторів, чия **загальна** виручка вища за загальну середню по всіх авторах

Створив таблиці за завданням

```sql
-- спочатку створюємо enum для статусів замовлень та типів книг
CREATE TYPE order_status AS ENUM ('нове', 'виконано', 'скасовано');

CREATE TYPE book_genre AS ENUM ('роман', 'фантастика', 'детектив', 'наукова', 'поезія', 'інше');

-- клієнти, тут пошта є унікальною і не нулем щоб можна було звязатися з клієнтом а телефон вже залишив без допису NOT NULL а то може клієнт не хоче ділитися своїм телефоном
CREATE TABLE customer (
    customer_id SERIAL PRIMARY KEY,
    customer_name VARCHAR(100) NOT NULL,
    phone VARCHAR(20),
    email VARCHAR(100) UNIQUE NOT NULL
);

-- автори, тут я так само для національності зробив без NOT NULL оскільки автор може це приховувати
CREATE TABLE author (
    author_id SERIAL PRIMARY KEY,
    author_name VARCHAR(100) NOT NULL,
    nationality VARCHAR(50),
    birth_year SMALLINT CHECK (birth_year > 1000 AND birth_year < 2010)
);

-- книги, ну тут ще додав обмеження на роки книг, щоб вже не прдавати чогось зовсім старого
CREATE TABLE book (
    book_id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    publish_year SMALLINT CHECK (publish_year > 1000 AND publish_year <= 2025),
    price NUMERIC(8, 2) NOT NULL CHECK (price >= 0),
    genre book_genre
);

-- тут в нас зв'язок книга - автор (багато до багатьох) тож реалізував як окрему таблицю з ключем (book_id, author_id)
CREATE TABLE book_author (
    book_id INT NOT NULL REFERENCES book(book_id),
    author_id INT NOT NULL REFERENCES author(author_id),
    PRIMARY KEY (book_id, author_id)
);

-- замовлення, ну тут в нас із особливого те що кожне замовлення коли робиться якщо не вказується дата то вона за умовчання іде як ось зараз зролене замовлення, і також коджне замовлення на початку за умовчанням отримує статус нове
CREATE TABLE order1 (
    order_id SERIAL PRIMARY KEY,
    customer_id INT NOT NULL REFERENCES customer(customer_id),
    order_date TIMESTAMP NOT NULL DEFAULT NOW(),
    status order_status NOT NULL DEFAULT 'нове'
);

-- позиції замовлення, тут додав обмеження на кількість книг в замовленні ну а не може ж бути 0 книг в замовленні :)
CREATE TABLE order_item (
    item_id SERIAL PRIMARY KEY,
    order_id INT NOT NULL REFERENCES order1(order_id),
    book_id INT NOT NULL REFERENCES book(book_id),
    quantity INT NOT NULL CHECK (quantity > 0)
);
```

А тепер переходимо до написання OLTP запитів
```sql
-- топ-10 книг за загальною виручкою за останні 12 місяців
-- тут ми рахуємо виручку для кожної книги це кількість помножити на ціну, джойнимо order_item з book щоб дістати ціну і з order1 щоб дістати дату, WHERE фільтрує тільки замовлення за останні 12 місяців, NOW() це поточна дата
-- GROUP BY по назві книги бо нам треба виручка окремо для кожної, ORDER BY DESC щоб найбільша виручка була зверху і LIMIT 10 бо нам треба топ-10
SELECT 
    b.title,
    SUM(oi.quantity * b.price) AS total_revenue
FROM order_item oi
    INNER JOIN book b USING (book_id)
    INNER JOIN order1 o USING (order_id)
WHERE o.order_date >= NOW() - INTERVAL '12 months'
GROUP BY b.title
ORDER BY total_revenue DESC
LIMIT 10;
```

```sql
-- для кожного клієнта знайти його замовлення з номером по порядку
-- тут використовуємо віконну функцію ROW_NUMBER() з лекції про агрегації, PARTITION BY розбиває замовлення по кожному клієнту окремо, ORDER BY order_date щоб нумерація йшла від найстарішого замовлення до найновішого,
-- тобто перше замовлення клієнта отримає номер 1, друге 2 і тд
SELECT 
    c.customer_name,
    o.order_id,
    o.order_date,
    o.status,
    ROW_NUMBER() OVER (PARTITION BY c.customer_id ORDER BY o.order_date) AS order_number
FROM order1 o
    INNER JOIN customer c USING (customer_id);
```

```sql
-- знайти авторів, чия середня виручка на книгу вища за середню по всіх авторах
-- використовуємо With бо нам потрібно двічі використати один і той же результат, спочатку в With рахуємо середню виручку на книгу для кожного автора, ділимо загальну виручку на кількість книг автора щоб отримати середню на книгу, а потім у головному запиті порівнюємо
-- кожного автора із середньою по всіх, підзапит SELECT AVG рахує середнє по всіх авторах
WITH author_revenue AS (
    SELECT 
        a.author_name,
        SUM(oi.quantity * b.price) / COUNT(DISTINCT b.book_id) AS avg_revenue_per_book
    FROM author a
        INNER JOIN book_author ba USING (author_id)
        INNER JOIN book b USING (book_id)
        INNER JOIN order_item oi USING (book_id)
    GROUP BY a.author_id, a.author_name
)
SELECT author_name, avg_revenue_per_book
FROM author_revenue
WHERE avg_revenue_per_book > (SELECT AVG(avg_revenue_per_book) FROM author_revenue)
ORDER BY avg_revenue_per_book DESC;
```

```sql
-- знайти авторів, чия загальна виручка вища за загальну середню по всіх авторах
-- майже те саме що і попередній але тут не середня на книгу а просто загальна виручка автора, тобто якщо автор написав багато книг і всі продавались то він матиме більшу загальну виручку, різниця між 3 і 4 запитом -- третій враховує скільки книг написав автор 
-- а четвертий ні :D
WITH author_total AS (
    SELECT 
        a.author_name,
        SUM(oi.quantity * b.price) AS total_revenue
    FROM author a
        INNER JOIN book_author ba USING (author_id)
        INNER JOIN book b USING (book_id)
        INNER JOIN order_item oi USING (book_id)
    GROUP BY a.author_id, a.author_name
)
SELECT author_name, total_revenue
FROM author_total
WHERE total_revenue > (SELECT AVG(total_revenue) FROM author_total)
ORDER BY total_revenue DESC;
```
